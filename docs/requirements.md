# 需求文档

## 项目概述

PDF 工具箱 — 浏览器端 + Node.js 后端的 PDF 处理工具。支持合并、压缩、水印、转图片四个核心功能，采用三栏编辑器布局。

## 功能需求

### F1 左侧边栏 — 文件管理

| ID | 需求 | 优先级 |
|---|---|---|
| F1.1 | 搜索框：输入关键字实时过滤已打开的文件名 | P1 |
| F1.2 | 打开文件按钮：触发系统文件选择器，支持多选 .pdf | P0 |
| F1.3 | 文件列表：已打开文件以卡片形式展示 | P0 |
| F1.4 | 文件卡片：显示文件名、大小、页数，hover 显示移除按钮 | P0 |
| F1.5 | 复选框多选：每个文件卡片前有复选框，用于合并 | P0 |
| F1.6 | 选中高亮：点击文件名在查看器预览，卡片高亮 | P0 |
| F1.7 | 设置按钮：底部固定，点击打开设置面板 | P2 |

### F2 中间查看器 — PDF 浏览

| ID | 需求 | 优先级 |
|---|---|---|
| F2.1 | Canvas 渲染：用 PDF.js 渲染当前页到 Canvas | P0 |
| F2.2 | 翻页：上一页/下一页按钮 + 页码显示 + 输入跳转 | P0 |
| F2.3 | 缩放：缩小/放大按钮 + 百分比显示 + 预设缩放值 | P0 |
| F2.4 | 向左旋转 90° | P1 |
| F2.5 | 向右旋转 90° | P1 |
| F2.6 | 保存按钮：下载当前查看的 PDF（含旋转效果） | P1 |

### F3 右侧工具栏 — PDF 处理

| ID | 需求 | 优先级 |
|---|---|---|
| F3.1 | 搜索框：输入关键字过滤工具名称 | P2 |
| F3.2 | 合并 PDF：多选文件 → 调用后端 → 下载合并结果 | P0 |
| F3.3 | 压缩 PDF：选中文件 → 调用后端 → 下载 + 大小对比 | P0 |
| F3.4 | 添加水印：配置文字/透明度/旋转/字号/颜色/位置 → 调用后端 → 下载 | P0 |
| F3.5 | 转为图片：配置格式/缩放 → 前端渲染 → 下载 PNG/ZIP | P0 |
| F3.6 | 工具配置面板：选择工具后替代右边栏，有返回按钮 | P0 |
| F3.7 | 按钮禁用态：根据文件数合理禁用（合并需 ≥2） | P1 |

### F4 非功能需求

| ID | 需求 |
|---|---|
| F4.1 | 纯浏览器端文件预览，不上传到服务器（查看器使用前端 PDF.js） |
| F4.2 | PDF 处理通过后端 API，支持 100MB 以内文件 |
| F4.3 | 蓝灰白配色，现代化 UI |
| F4.4 | 亮色/暗色主题切换 (P2) |
| F4.5 | 响应式布局（小屏幕降级提示） |

## 用户场景

1. **场景 A**：用户拖入 3 份 PDF → 勾选 2 份 → 右侧点击合并 → 下载合并文件
2. **场景 B**：用户打开 1 份 PDF → 点击文件名在查看器浏览 → 发现太大 → 点击压缩 → 下载压缩版
3. **场景 C**：用户打开 PDF → 配置水印参数 → 点击添加 → 下载带水印的文件
4. **场景 D**：用户打开 PDF → 点击转图片 → 下载每页 PNG 的 ZIP 包
